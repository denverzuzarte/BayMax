> Built on top of Loop AI (`../anthropicc`) codebase

## 0. What to Keep vs Replace from anthropicc

**Keep:**
- Voice pipeline: STT (Google Speech Recognition), TTS (edge-tts), FFmpeg/pydub
- `detect_department()` — fast pre-filter before Claude call
- Emergency keyword detection — extend with more terms, keep short-circuit logic
- Session management — extend with user profile
- Twilio integration — extend for SMS post-visit feedback

**Replace:**
- `process_with_gemini()` → `process_with_claude()` using Anthropic SDK
- `get_network_status()` → extend to check PMJAY + insurance coverage
- System prompt → BayMax master prompt (Section 6)

**Delete entirely:**
- `hospital_rag.py`
- `hospital_index.faiss`
- `hospital_data.pkl`
- `network_status.json`

These were built for semantic name search — wrong tool for location-based lookup.
Google Places API replaces all of it with one API call.

## 4. Conversation State Machine

Max 3 clarification rounds. One question per turn. Never route on incomplete info.

```
STATE 1 — GREETING
  Plays hardcoded opening greeting (not AI generated)
  User describes symptoms → STATE 2

STATE 2 — CLARIFICATION (max 3 rounds)
  Ask ONE question per turn based on what is missing:
    "On a scale of 1 to 10, how much pain are you in?"
    "How long have you had this symptom?"
    "Is it getting worse quickly or gradually?"
    "Do you also have fever, vomiting, or difficulty breathing?"
    "Have you experienced this before?"

  pain >= 8 OR chest pain OR breathing difficulty → EMERGENCY STATE
  Enough context after <= 3 questions → STATE 3

EMERGENCY STATE
  Voice: "This sounds serious. Please call 112 immediately."
  UI: Nearest emergency hospitals from Google Places
  UI: Full emergency numbers panel
  END

STATE 3 — ROUTING
  quick_department_lookup(text)   → local keyword dict, ~0ms
  If no match → route_with_claude(symptom_history, user_profile)
  Fire parallel API calls: Places + PMJAY + CGHS
  → STATE 4

STATE 4 — RESULTS
  Voice: BayMax summarises in 2-3 sentences
  UI: Top 3 hospital/clinic cards with full detail
  → STATE 5

STATE 5 — FOLLOW-UP
  "Would you like directions, more options, or information about any of these?"
  Re-use last referenced hospital context
```

## 6. Parallel API Call Pattern

```python
import asyncio

async def get_full_context(department, lat, lng, user_profile):
    results = await asyncio.gather(
        search_facilities(lat, lng, department),
        check_pmjay_eligibility(user_profile.get("pmjay_id")),
        get_cost_estimate(department),
        return_exceptions=True
    )
    facilities, pmjay, costs = results
    return {
        "facilities": facilities if not isinstance(facilities, Exception) else [],
        "pmjay":      pmjay      if not isinstance(pmjay, Exception)      else {"eligible": False},
        "costs":      costs      if not isinstance(costs, Exception)       else None
    }
```

## 7. Master Symptom Router Prompt (`prompts/symptom_router.txt`)

Load once at startup. Inject variables at call time. Expect JSON back only.

```
You are BayMax, a calm and caring medical assistant helping Indian patients
find the right doctor or hospital. You do NOT diagnose — you route.

Your voice is slow, soft, and reassuring. You never panic the patient.
You always remind them you are not a doctor.

PATIENT PROFILE:
- Name: {name}
- City: {city}
- Insurance: {insurance_provider} / {insurance_plan}
- PMJAY eligible: {pmjay_eligible}

SYMPTOM HISTORY (full conversation so far):
{symptom_history}

AVAILABLE DEPARTMENTS IN NEARBY FACILITIES:
{available_departments}

DETERMINE ALL OF THE FOLLOWING:

1. URGENCY
   - emergency: life-threatening — chest pain, stroke signs, severe breathing
     difficulty, unconsciousness, heavy bleeding, pain >= 8/10
   - urgent: needs attention within 24 hours
   - routine: can book in coming days

2. DEPARTMENT
   Most appropriate from available list. If no exact match, closest available.

3. DOCTOR TYPE
   Specialist within that department.
   If a general physician at a clinic suffices, say so explicitly.

4. CLARIFICATION
   If insufficient info to route confidently, set routing_complete to false.
   Provide exactly ONE short plain-language question. Assume low medical literacy.

5. LIKELY CONDITIONS
   2-3 possibilities. Always frame as possibilities, never diagnosis.
   Example phrasing: "This could be related to X, Y, or Z."

RULES:
- If urgency is emergency → voice_response must be ONLY:
  "This sounds serious. Please call 112 immediately." Nothing else.
- Never suggest specific medication
- Never confirm a diagnosis
- Always end with reminder to consult a professional
- Respond in the same language the patient used
- voice_response is for TTS: max 3 sentences, conversational, no markdown

RESPOND IN JSON ONLY. No preamble, no explanation outside the JSON:
{
  "urgency": "emergency | urgent | routine",
  "department": "string",
  "doctor_type": "string",
  "likely_conditions": ["string", "string"],
  "routing_complete": true | false,
  "next_clarifying_question": "string | null",
  "voice_response": "string"
}
```

1. Open BayMax on phone — installed via PWA
2. BayMax: *"Hi, I am Baymax, your personal healthcare companion. I am not a doctor. I will help you find the right one."*
3. Signup: Priya, Mumbai, Star Health Insurance
4. Say in Hindi: **"मुझे सीने में दर्द हो रहा है"**
5. Emergency flow: *"This sounds serious. Please call 112 immediately."* + hospital cards
6. Reset → **"I've had a headache for 3 days and some nausea"**
7. BayMax asks: *"On a scale of 1 to 10, how much pain are you in?"*
8. User: "About a 4, getting worse" → routes to Neurology
9. 3 hospital cards: cost ₹600–900, PMJAY badge, one-tap call
10. Show emergency page — always reachable

**Judge Q&A:**
- *Who benefits?* Uninsured/underinsured Indians navigating a fragmented system
- *What could go wrong?* Wrong routing → every response ends with "please confirm with a professional"
- *How does it help without deciding for people?* It routes, never diagnoses. The patient makes the call. Literally.
